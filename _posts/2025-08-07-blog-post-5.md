### **The Problem**  
Static maps are easy to build, but many applications like tracking sensor data, live assets, or fieldwork updates require maps that refresh without user interaction. Pushing updates via WebSockets or dedicated APIs is overkill for simple use cases where data changes every few minutes (not seconds) and where simplicity matters more than real-time precision.  

A practical solution is to **host a GeoJSON file on S3 and periodically fetch updates in the browser**. This approach works well for:  
- Field teams submitting GPS tracks.  
- Sensor networks reporting hourly/daily.  
- Crowdsourced datasets with intermittent edits.  

Here’s an overview of how I built mine.  

---  

### **Architecture**  
1. **Data Pipeline** (AKA the backend):  
   - Generates/updates a GeoJSON file.  
   - Uploads it to S3 (e.g., `s3://my-bucket/data/latest.geojson`).  

2. **Frontend**:  
   - Leaflet map loads the initial GeoJSON from S3.  
   - JavaScript checks for updates at a set interval (e.g., 5 minutes).  
   - If the file changes, the map refreshes.  

3. **AWS Setup**:  
   - S3 bucket with CORS enabled.  
   - Optional: CloudFront CDN for faster global access.  

---  

### **Step 1: Configuring S3 for GeoJSON Hosting**  
First, ensure your bucket allows public reads (or signed requests, if private):  

1. **Bucket Policy** (for public access):  
```json
{
  "Version": "2025-08-06",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/data/*"
    }
  ]
}
```  

2. **CORS Configuration** (required for browser requests):  

```html
<CORSConfiguration>
  <CORSRule>
    <AllowedOrigin>*</AllowedOrigin>
    <AllowedMethod>GET</AllowedMethod>
    <MaxAgeSeconds>3000</MaxAgeSeconds>
  </CORSRule>
</CORSConfiguration>
```  

3. **Uploading the GeoJSON**:  
Use the AWS CLI to sync updates:  
```bash
aws s3 cp ./latest.geojson s3://my-bucket/data/latest.geojson --acl public-read
```  

---  

### **Step 2: Building the Leaflet Map**  
The frontend needs to:  
1. Load the initial GeoJSON.  
2. Poll for changes (using file `Last-Modified` timestamps).  
3. Update layers without refreshing the page.  

#### **HTML Template**  
```html
<!DOCTYPE html>
<html>
<head>
  <title>Auto-Updating Map</title>
  <link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" />
  <style>
    #map { height: 600px; width: 100%; }
  </style>
</head>
<body>
  <div id="map"></div>
  <script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
  <script src="app.js"></script>
</body>
</html>
```  

#### **JavaScript (app.js)**  
```javascript
// Initialize map
const map = L.map('map').setView([50, -0.1], 13);
L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
  attribution: '© OpenStreetMap'
}).addTo(map);

// Global variable to store the current GeoJSON layer
let currentGeoJsonLayer = null;
let lastModified = null;

// Fetch GeoJSON from S3
async function fetchGeoJson() {
  const url = 'https://my-bucket.s3.amazonaws.com/data/latest.geojson';
  
  try {
    // Check if the file has been modified
    const headResp = await fetch(url, { method: 'HEAD' });
    const newLastModified = headResp.headers.get('last-modified');
    
    if (newLastModified !== lastModified) {
      // Fetch the full GeoJSON if modified
      const resp = await fetch(url);
      const geojson = await resp.json();
      
      // Remove the old layer if it exists
      if (currentGeoJsonLayer) {
        map.removeLayer(currentGeoJsonLayer);
      }
      
      // Add the new layer
      currentGeoJsonLayer = L.geoJSON(geojson, {
        style: { color: '#0066ff', weight: 2 },
        onEachFeature: (feature, layer) => {
          if (feature.properties.popupContent) {
            layer.bindPopup(feature.properties.popupContent);
          }
        }
      }).addTo(map);
      
      // Update last modified timestamp
      lastModified = newLastModified;
      console.log('Map updated at:', new Date());
    }
  } catch (err) {
    console.error('Failed to fetch GeoJSON:', err);
  }
}

// Poll for updates every 5 minutes
fetchGeoJson(); // Load immediately
setInterval(fetchGeoJson, 5 * 60 * 1000);
```  

#### **Key Details**:  
- **HEAD Requests**: Checking `Last-Modified` before downloading the full file saves bandwidth.  
- **Layer Management**: Clearing the old layer prevents duplicates.  
- **Error Handling**: Silent fails (with console logs) avoid breaking the map if S3 is temporarily unreachable.  

---  

### **Step 3: Optimizing Performance**  
1. **Reduce File Size**:  
   - Simplify geometries (e.g., with `turf.simplify()` before uploading).  
   - Use `ndjson` (Newline-Delimited GeoJSON) for large files, parsing incrementally.  

2. **Cache Control**:  
   - Set `Cache-Control: max-age=300` on the S3 object to allow browser caching.  
   - Use CloudFront to edge-cache the GeoJSON.  

3. **Delta Updates**:  
   For frequent updates, append-only logs (e.g., `updates.ndjson`) can reduce transfer size:  
   ```javascript
   // Fetch only new features since last check
   async function fetchUpdates(timestamp) {
     const url = `https://my-bucket.s3.amazonaws.com/data/updates.ndjson?since=${timestamp}`;
     const resp = await fetch(url);
     const text = await resp.text();
     text.split('\n').forEach(line => {
       if (line) {
         const feature = JSON.parse(line);
         currentGeoJsonLayer.addData(feature);
       }
     });
   }
   ```

### **Step 4: Securing the Data (If Needed)**  
For private data, generate **pre-signed S3 URLs** (valid for a short time) via a backend:  

```python
# Flask example for generating pre-signed URLs
from flask import Flask, jsonify
import boto3
from datetime import datetime, timedelta

app = Flask(__name__)
s3 = boto3.client('s3')

@app.route('/get-geojson-url')
def get_geojson_url():
    url = s3.generate_presigned_url(
        'get_object',
        Params={'Bucket': 'my-bucket', 'Key': 'data/latest.geojson'},
        ExpiresIn=3600  # 1-hour expiry
    )
    return jsonify({'url': url})
```  

Then modify the frontend to use this endpoint:  
```javascript
async function fetchGeoJson() {
  const { url } = await fetch('/get-geojson-url').then(res => res.json());
  // Rest of the logic remains the same
}
```

### **Final Notes**  
**Reasons this worked for my project's needs**:  
- **Simplicity**: No databases, WebSockets, or complex backends. Just static files.  
- **Cost-Effective**: S3 costs pennies for moderate traffic.  
- **Scalable**: CloudFront handles global traffic spikes effortlessly.  

**Limitations**:  
- Not truly real-time (polling delays).  
- No change history (unless you version files).  

**Alternatives**:  
- For real-time needs, consider MQTT or Server-Sent Events (SSE).  
- For history, use S3 versioning or a database like PostGIS.  

This pattern fits projects where "near-real-time" is good enough, and minimal infrastructure is a priority. It’s been reliable for field data collection, IoT dashboards, and intermittently updated geospatial data.
